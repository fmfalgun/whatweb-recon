> WHAT IS WHATWEB

WhatWeb identifies web technologies including CMS, blogging platforms, stats/analytics packages, JavaScript libraries, web servers, and embedded devices. It has over 1800 plugins.

> WHY THIS SCRIPT

  • Standalone — no shared database
  • Auto-falls back to HTTP fingerprint if WhatWeb binary missing
  • Highlights EOL PHP, exposed xmlrpc, admin panels
  • Community Tech Board via GitHub Issues submission
  • 24h SQLite cache — offline-first

> USAGE

# scan a domain (auto-expands to https:// and https://www.) python3 whatweb-recon.py -d nmap.org # scan a list of URLs python3 whatweb-recon.py --input urls.txt # write JSON output python3 whatweb-recon.py -d nmap.org -o result.json # override aggression level (1=stealthy, 3=aggressive, 4=heavy) python3 whatweb-recon.py -d nmap.org --aggression 1 # submit to community Tech Board python3 whatweb-recon.py -d nmap.org --submit # bypass cache python3 whatweb-recon.py -d nmap.org --no-cache

> AGGRESSION LEVELS

WhatWeb uses aggression to control how intrusively it probes a target:

# Level 1 — Stealthy One HTTP request per URL. Minimal footprint. Use for CDN/WAF-protected targets or large URL sets. Reads headers and homepage HTML only. # Level 3 — Aggressive Fetches additional pages (/admin, /login, /readme.html) and checks X-Powered-By. Default for most targets. Detects more plugins and CMS versions. # Level 4 — Heavy Dictionary-attack style: many HTTP requests per URL. Use sparingly. Can trigger WAF blocks. Auto-selected only on explicit --aggression 4. # Auto-selection heuristic > 30 URLs → level 1 (stealthy) Otherwise → level 3 (aggressive)
